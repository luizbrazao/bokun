<task type="checkpoint:human-action">
  <name>Task 1: Connect repo to Render, set environment secrets, trigger first deploy</name>
  <action>
This task requires human interaction with the Render and Convex dashboards. Claude cannot automate dashboard interactions.

STEP 1 — Generate Convex Production Deploy Key:
1. Open Convex Dashboard > your project > Settings > Deploy Keys
2. Click "Generate New Key" — select type: Production
3. Copy the full key (format: prod:qualifier-name-123|eyJ2...0=)
4. Keep it ready — you will paste it as CONVEX_DEPLOY_KEY in Render

STEP 2 — Connect Render to the Repository via Blueprint:
1. Open Render Dashboard > New > Blueprint
2. Connect your GitHub account if not already connected
3. Select the Bokun repository
4. Render will detect render.yaml and show two services: bokun-bot-api and bokun-bot-dashboard
5. Click "Apply Blueprint" — do NOT trigger a deploy yet

STEP 3 — Fill all sync:false secrets BEFORE the first deploy.
For bokun-bot-api service, go to Environment and fill in:
- CONVEX_URL: Convex Dashboard > Deployment > Settings > Deployment URL (e.g., https://xxx.convex.cloud)
- CONVEX_SITE_URL: Convex Dashboard > Deployment > Settings > Site URL
- CONVEX_DEPLOY_KEY: The production deploy key generated in Step 1
- WHATSAPP_APP_SECRET: Meta Developer Console > App > App Secret
- WHATSAPP_VERIFY_TOKEN: Meta Developer Console > App > WhatsApp > Webhook > Verify Token
- BOKUN_APP_CLIENT_ID: Bokun App Marketplace > Your App > Credentials
- BOKUN_APP_CLIENT_SECRET: Bokun App Marketplace > Your App > Credentials
- BOKUN_OAUTH_REDIRECT_URI: https://{your-render-service}.onrender.com/oauth/callback
- BOKUN_ACCESS_KEY: Bokun vendor dashboard API credentials (if using HMAC auth)
- BOKUN_SECRET_KEY: Bokun vendor dashboard API credentials (if using HMAC auth)
- OPENAI_API_KEY: OpenAI Dashboard > API Keys (global fallback)
- SENTRY_DSN: Sentry Dashboard > Project > Client Keys (optional; skip if not using Sentry)
- STRIPE_SECRET_KEY: Leave empty for now — required for Phase 3
- STRIPE_WEBHOOK_SECRET: Leave empty for now — required for Phase 3

For bokun-bot-dashboard service, fill in:
- VITE_CONVEX_URL: Same value as CONVEX_URL above (must have VITE_ prefix)

STEP 4 — Trigger first manual deploy (AFTER all env vars are set):
1. Go to bokun-bot-api > Manual Deploy > Deploy latest commit
2. Watch the deploy log — you should see three phases:
   - Build phase: npm install
   - Pre-deploy phase: npm run convex:deploy (Convex schema deployed to production)
   - Start phase: npm start
3. If convex:deploy fails with "browser login required" → CONVEX_DEPLOY_KEY is not set correctly
4. Wait for "Deploy live" status before proceeding to Task 2

<task type="checkpoint:human-verify">
  <name>Task 2: Verify health endpoint and HMAC webhook signatures in production</name>
  <action>
Run the following verification commands against the live production URL. Replace {RENDER_URL} with the actual bokun-bot-api URL shown in Render Dashboard (e.g., https://bokun-bot-api-xxxx.onrender.com).

Check 1 — Health endpoint:
  curl -s https://{RENDER_URL}/health | jq .
  Expected: JSON with status, version, uptime fields and HTTP 200

Check 2 — WhatsApp webhook GET verification:
  curl -s "https://{RENDER_URL}/whatsapp/webhook?hub.mode=subscribe&hub.verify_token=YOUR_WHATSAPP_VERIFY_TOKEN&hub.challenge=test123"
  Expected: returns test123 (challenge echoed back). 403 means WHATSAPP_VERIFY_TOKEN is wrong.

Check 3 — Meta HMAC rejection (confirms WHATSAPP_APP_SECRET is active):
  curl -s -w "\nHTTP %{http_code}" -X POST https://{RENDER_URL}/whatsapp/webhook \
    -H "Content-Type: application/json" \
    -H "X-Hub-Signature-256: sha256=invalidsignature" \
    -d '{"test": true}'
  Expected: HTTP 401 or 403 (invalid signature — HMAC validation working correctly)

Check 4 — Bokun HMAC rejection (confirms BOKUN_APP_CLIENT_SECRET is active):
  curl -s -w "\nHTTP %{http_code}" -X POST https://{RENDER_URL}/bokun/webhook \
    -H "Content-Type: application/json" \
    -H "x-bokun-hmac: invalidsignature" \
    -d '{"test": true}'
  Expected: HTTP 400, 401, or 403 (invalid signature)

Check 5 — Convex production active (no code steps):
  Open Convex Dashboard > your project — confirm production deployment shows current functions with no schema warnings.

All 5 checks passing satisfies DEPLOY-04.
  </action>
  <verify>
    All 5 checks pass:

    - Convex Dashboard shows active production deployment
  </verify>
  <done>DEPLOY-04 satisfied: health endpoint 200, Meta HMAC validation active, Bokun HMAC validation active, Convex production deployed and current</done>
</task>
