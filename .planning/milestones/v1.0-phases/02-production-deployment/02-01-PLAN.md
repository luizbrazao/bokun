Create render.yaml at the repo root. Requirements from locked decisions in 02-CONTEXT.md:

**CRITICAL constraints (do not deviate):**
- `sync: false` is valid ONLY at service-level envVars — NOT inside envVarGroups blocks (Render silently drops them from envVarGroups). Use per-service envVars with `sync: false` for all secrets.
- Backend plan MUST be `starter` (not `free`) — `preDeployCommand` does not run on Free tier
- `startCommand` must be `npm start` (not the raw node command) — npm start already includes the Sentry preload flag
- React static site needs the catch-all rewrite rule or React Router 404s on direct URL access

**render.yaml structure:**

```yaml
# render.yaml
# Render Blueprint — deploys both services from a single git push.
# Docs: https://render.com/docs/blueprint-spec

services:
  # ── Node.js Backend (Web Service) ────────────────────────────────────────
  - type: web
    name: bokun-bot-api
    runtime: node
    plan: starter
    region: frankfurt
    rootDir: .
    buildCommand: npm install
    preDeployCommand: npm run convex:deploy
    startCommand: npm start
    healthCheckPath: /health
    autoDeploy: true
    envVars:
      - key: NODE_ENV
        value: production
      - key: SENTRY_ENVIRONMENT
        value: production
      - key: LOG_LEVEL
        value: info
      - key: BOKUN_BASE_URL
        value: https://api.bokun.io
      - key: OPENAI_MODEL
        value: gpt-4o-mini
      # Secrets — values entered manually in Render Dashboard after Blueprint sync
      - key: CONVEX_URL
        sync: false
      - key: CONVEX_SITE_URL
        sync: false
      - key: CONVEX_DEPLOY_KEY
        sync: false
      - key: WHATSAPP_APP_SECRET
        sync: false
      - key: WHATSAPP_VERIFY_TOKEN
        sync: false
      - key: BOKUN_APP_CLIENT_ID
        sync: false
      - key: BOKUN_APP_CLIENT_SECRET
        sync: false
      - key: BOKUN_OAUTH_REDIRECT_URI
        sync: false
      - key: BOKUN_ACCESS_KEY
        sync: false
      - key: BOKUN_SECRET_KEY
        sync: false
      - key: OPENAI_API_KEY
        sync: false
      - key: SENTRY_DSN
        sync: false
      # Phase 3 placeholders (Stripe billing — not yet wired in app code)
      - key: STRIPE_SECRET_KEY
        sync: false
      - key: STRIPE_WEBHOOK_SECRET
        sync: false

  # ── React Dashboard (Static Site) ────────────────────────────────────────
  - type: web
    name: bokun-bot-dashboard
    runtime: static
    plan: free
    region: frankfurt
    rootDir: frontend
    buildCommand: npm install && npm run build
    staticPublishPath: dist
    autoDeploy: true
    routes:
      - type: rewrite
        source: /*
        destination: /index.html
    envVars:
      - key: VITE_CONVEX_URL
        sync: false
```

Write exactly this structure. Do not use `envVarGroups` at the top level (sync:false constraint makes per-service envVars the correct approach).
