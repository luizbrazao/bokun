- Export `createRequestLogger(bindings)` factory function that returns `rootLogger.child(bindings)`
- The bindings type must include: `tenantId: string`, `channel: string`, `requestId: string`, and optional `correlationId?: string`, `messageId?: string`, `providerMessageId?: string`, `event?: string`
  - `messageId` is the internal/dedup message ID
  - `providerMessageId` is the WhatsApp message ID from the Meta webhook (msg.messageId in server.ts) â€” include when available per locked decision; pino silently drops undefined fields so pass as undefined when not applicable (health, OAuth routes)
- For routes without a message context (health, OAuth), omit both messageId and providerMessageId from bindings â€” pass as undefined or omit the field

The file must be ES Module compatible (no require). pino v10 ships with ESM support natively.

```typescript
// src/lib/logger.ts
import pino from "pino";

export const rootLogger = pino({
  level: process.env.LOG_LEVEL ?? "info",
  timestamp: pino.stdTimeFunctions.isoTime,
});

export function createRequestLogger(bindings: {
  tenantId: string;
  channel: string;
  requestId: string;
  correlationId?: string;
  messageId?: string;
  providerMessageId?: string;
  event?: string;
}) {
  return rootLogger.child(bindings);
}

export default rootLogger;
```
