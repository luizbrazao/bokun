**File 1: `src/bokun/availabilityToOptionMap.test.ts`**

Test the `availabilityToOptionMap` function from `./availabilityToOptionMap.ts`. Cases:

1. **Empty availabilities** → returns OptionMap with `options: []`
2. **Timezone propagated** → `tz` field in returned OptionMap equals the passed `tz` argument ("America/Sao_Paulo"), NOT hardcoded "Europe/Madrid"
3. **Default tz fallback** → when `tz` is omitted, defaults to "Europe/Madrid"
4. **Filters zero-count availabilities** → availabilities with `availabilityCount: 0` are excluded from options
5. **Sorts by date then startTime** → two availabilities on same date, later startTime second
6. **Respects limit** → with 5 availabilities and `limit: 3`, options has length 3
7. **optionId sequence** → options[0].optionId === "1", options[1].optionId === "2"
