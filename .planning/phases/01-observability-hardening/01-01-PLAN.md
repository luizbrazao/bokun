Create src/lib/logger.ts with:
- Root pino logger instance (singleton) configured with:
  - `level: process.env.LOG_LEVEL ?? "info"`
  - `timestamp: pino.stdTimeFunctions.isoTime`
- Export `rootLogger` as the default export and named export
- Export `createRequestLogger(bindings)` factory function that returns `rootLogger.child(bindings)`
- The bindings type must include: `tenantId: string`, `channel: string`, `requestId: string`, and optional `correlationId?: string`, `messageId?: string`, `event?: string`
- For routes without a message context (health, OAuth), omit messageId from bindings — pino silently drops undefined fields, so pass `messageId: undefined` safely

The file must be ES Module compatible (no require). pino v10 ships with ESM support natively.

```typescript
// src/lib/logger.ts
import pino from "pino";

export const rootLogger = pino({
  level: process.env.LOG_LEVEL ?? "info",
  timestamp: pino.stdTimeFunctions.isoTime,
});

export function createRequestLogger(bindings: {
  tenantId: string;
  channel: string;
  requestId: string;
  correlationId?: string;
  messageId?: string;
  event?: string;
}) {
  return rootLogger.child(bindings);
}

export default rootLogger;
```
